-----------------
1.  소개
-----------------
NFS는 FUSE와 소켓통신을 통한 네트워크 파일 시스템이다. 이 시스템은 FUSE 2.9.6위에서 실행되고 C++로 컴파일 할 수 있다. 또한 protobuf를 사용하여 메세지를 패킹하기 때문에 protobuf3이상의 버젼을 설치해야한다.

NFS는 마운트 시킬 디렉토리의 서버 IP주소와 서버 프로그램의 포트번호, 클라이언트의 마운트포인트를 입력하여 서버 디렉토리를 마운트할 수 있다.

클라이언트의 마운트 포인트에서 서버 디렉토리에 파일 관련 시스템콜 및 명령어를 실행할 수 있다.

6.  설치 및 실행
--------------------
프로그램을 설치하기 위해선 먼저 protobuf를 컴파일 한다.

	proton —cpp_out=./ ./nfs_data.cc

다음 클라이언트인 fuse를 컴파일 한다.

	g++ -W fusexmp.cc nfs_data.pb.cc -o client -D_FILE_OFFSET_BITS=64 -DFUSE_USE_VERSION=26 -lfuse -lprotobuf

서버 프로그램 컴파일 방법이다.

	g++ server.cc nfs_data.pb.cc -o server -lprotobuf

컴파일이 완료되면 서버프로그램을 먼저 실행 시킨다.

	./server [포트번호]

서버가 정상적으로 실행되면 클라이언트를 실행시켜 마운트한다.

	./client [IP주소] [포트번호] [마운트포인트]

---------------
7.  한계 및 버그
---------------
NFS에는 현재 몇몇의 발견된 버그와 한계가 존재한다.

    - 기타 에디터를 통해 파일 생성에 문제가 있다.
    - 통신 연결이 불안정하면 마운트가 끊길 수 있다.

최초 버젼이기 때문에 많은 버그와 한계가 존재하기 때문에 발견한다면 메일로 리포트할 수 있다.
